### 2. ✅ Atribuição Automática ao Clicar no Work Item
**NOVO**: Quando clicar em um work item sem assignee, pergunta se quer se atribuir

**Fluxo**:
1. Usuário clica no work item
2. Se não tem assignee → Pergunta: "Deseja se atribuir como desenvolvedor?"
3. Se sim → Atribui o usuário e marca como desenvolvedor
4. Se o status for 'backlog' ou 'ready' → Pergunta se quer iniciar
5. Abre o work item para visualização

**Arquivo**: `src/services/WorkItemService.ts`
```typescript
async open(workItem: WorkItem): Promise<void> {
    // Se não tem assignee, perguntar se quer se atribuir
    if (!workItem.assignee_id) {
        const shouldAssign = await vscode.window.showInformationMessage(
            `Deseja se atribuir como desenvolvedor de "${workItem.title}"?`,
            'Sim, Me Atribuir',
            'Não, Apenas Visualizar'
        );
        
        if (shouldAssign === 'Sim, Me Atribuir') {
            await this.assignToMe(workItem.id);
            workItem.assignee_id = user.id;
        }
    }
    // ...
}
```

## Endpoints Esperados no Backend

### 1. Work Items
```
GET /api/v1/work-items?project_id={projectId}
```
Retorna: Array de todos os work items do projeto

### 2. Atribuir Work Item
```
PATCH /api/v1/work-items/{workItemId}
Body: { "assignee_id": "user-uuid" }
```

### 3. Specification
```
GET /api/v1/specifications?project_id={projectId}
```
Retorna: Array ou objeto com specification

### 4. Architecture
```
GET /api/v1/architectures?project_id={projectId}
```
Retorna: Array ou objeto com architecture

## Como Funciona Agora

### Fluxo Completo

1. **Login** → Usuário faz login
2. **Selecionar Projeto** → Escolhe um projeto
3. **Ver Árvore**:
   ```

   📝 Work Items          ← Expanda para ver todos
   ```
